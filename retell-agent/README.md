# Retell AI Voice Agent Configuration\n\nThis folder contains all configuration files, templates, and documentation for the LEO voice agent powered by Retell.AI.\n\n## Overview\n\nLEO is designed to handle property management inquiries, tenant screening, maintenance requests, and general leasing support. The voice agent uses natural language processing to understand tenant and prospect needs and provide appropriate responses while guiding them toward booking tours and capturing lead information.\n\n## Files Structure\n\n### Core Configuration Files\n\n#### `LeoVoiceAgent.json`\n- **Purpose**: Main Retell.AI agent configuration file\n- **Contains**: Agent settings, voice parameters, and system configuration\n- **Usage**: Import directly into Retell.AI dashboard\n- **Format**: JSON configuration for agent deployment\n\n#### `PROMPT - Leo Voice Agent.md`\n- **Purpose**: Complete conversation framework and agent behavior instructions\n- **Contains**: \n  - Role definition and personality traits\n  - Detailed conversation flow (10 steps from greeting to wrap-up)\n  - Communication style guidelines\n  - Function usage instructions\n  - Sample dialogues and escalation procedures\n- **Usage**: Primary prompt engineering document for voice agent behavior\n- **Format**: Structured markdown with conversation flow logic\n\n### Knowledge Base and Data\n\n#### `Lakeside Apartments - MOCK DATA SET.md`\n- **Purpose**: Complete property information dataset for testing and development\n- **Contains**: \n  - Property details, amenities, and location information\n  - Voice-safe responses and escalation scripts\n  - FAQ sections and canonical questions\n  - Test probes for validation\n- **Usage**: Example knowledge base for Lakeside Apartments property\n- **Format**: Voice-optimized markdown with chunking guidance\n\n#### `Knowledge Base Template.md`\n- **Purpose**: Standardized template for creating property knowledge bases\n- **Contains**: \n  - Structured sections for property information\n  - Placeholder fields for easy customization\n  - Best practices for voice-safe content\n  - Escalation guidelines and test scenarios\n- **Usage**: Template for creating new property knowledge bases\n- **Format**: Reusable markdown template\n\n### Documentation and Guidelines\n\n#### `Best Practices Retell knowledge base.md`\n- **Purpose**: Guidelines for optimizing content for Retell.AI\n- **Contains**: \n  - File format recommendations\n  - Content structuring best practices\n  - Chunking and embedding guidelines\n  - Testing and update procedures\n- **Usage**: Reference guide for knowledge base preparation\n- **Format**: Best practices summary with quick reference table\n\n#### `LVA - Lakeside.rtf`\n- **Purpose**: Rich text format version of agent configuration\n- **Contains**: Formatted version of agent setup and configuration\n- **Usage**: Alternative format for documentation and sharing\n- **Format**: Rich Text Format (RTF)\n\n### Configuration Files\n\n#### `.gitignore`\n- **Purpose**: Defines files and patterns to exclude from version control\n- **Contains**: \n  - Sensitive configuration files (API keys, secrets)\n  - Temporary and backup files\n  - Environment-specific configurations\n- **Usage**: Maintains security and clean repository\n\n## Configuration Guide\n\n### Agent Setup Process\n\n1. **Import Base Configuration**\n   - Upload `LeoVoiceAgent.json` to Retell.AI dashboard\n   - Configure voice settings and conversation parameters\n   - Set up webhook endpoints for N8N integration\n\n2. **Configure Conversation Logic**\n   - Review and customize `PROMPT - Leo Voice Agent.md`\n   - Adapt conversation flows for specific property needs\n   - Set up escalation procedures and function calls\n\n3. **Prepare Knowledge Base**\n   - Use `Knowledge Base Template.md` to create property-specific content\n   - Follow guidelines in `Best Practices Retell knowledge base.md`\n   - Structure content for optimal voice retrieval\n   - Reference `Lakeside Apartments - MOCK DATA SET.md` as example\n\n4. **Testing and Validation**\n   - Test conversation flows with sample scenarios\n   - Validate knowledge base retrieval accuracy\n   - Ensure proper escalation handling\n   - Monitor voice quality and response timing\n\n### Voice Agent Capabilities\n\n#### Core Functions\n- **Lead Qualification**: Capture contact information and rental preferences\n- **Tour Scheduling**: Check availability and book property tours\n- **Information Provision**: Answer questions about amenities, location, and policies\n- **Escalation Management**: Route complex queries to human agents\n- **Follow-up Automation**: Trigger email confirmations and nurture sequences\n\n#### Conversation Flow\nThe agent follows a structured 10-step process:\n1. Greeting & Filtering\n2. Rapport & Intent Discovery\n3. Lead Qualification\n4. Preference Discovery\n5. Contact Info Capture\n6. Scheduling Eligibility Check\n7. Calendar Coordination\n8. Appointment Booking\n9. Wrap-Up & Follow-Up\n10. Escalation or Exit\n\n### Integration Points\n\n#### Webhook Integration\n- **N8N Workflows**: Real-time data processing and CRM integration\n- **Lead Processing**: Automatic lead scoring and routing\n- **Email Automation**: Confirmation and follow-up sequences\n- **Calendar Systems**: Tour scheduling and availability checking\n\n#### Function Calls\n- `check_availability`: Query calendar for tour slots\n- `schedule_appointment`: Book confirmed tours\n- `send_email_summary`: Deliver property information\n- `log_lead_data`: Capture prospect information\n- `end_call`: Graceful conversation completion\n\n### Best Practices\n\n#### Content Optimization\n- Use voice-safe language and natural phrasing\n- Structure information in 500-token chunks with 100-token overlap\n- Include canonical questions and test probes\n- Separate voice-safe content from internal-only information\n\n#### Conversation Design\n- Keep responses concise (1-2 sentences maximum)\n- Ask one clear question at a time\n- Wait for complete caller responses before proceeding\n- Maintain consistent, friendly personality\n- Always confirm contact information spelling\n\n#### Security and Compliance\n- Never quote exact pricing or lease terms\n- Escalate complex policy questions to human agents\n- Handle PII data according to privacy regulations\n- Maintain conversation logs for quality assurance\n\n## Deployment Process\n\n### Development Environment\n1. Test agent configuration with sample data\n2. Validate conversation flows and escalation procedures\n3. Ensure proper integration with N8N webhooks\n4. Verify knowledge base retrieval accuracy\n\n### Staging Environment\n1. Deploy to staging Retell.AI environment\n2. Conduct end-to-end testing with realistic scenarios\n3. Validate email and calendar integrations\n4. Performance testing under load\n\n### Production Deployment\n1. Import final configuration to production environment\n2. Set up monitoring and alerting\n3. Configure backup and disaster recovery\n4. Train support team on agent capabilities\n\n## Monitoring and Maintenance\n\n### Key Metrics\n- Conversation completion rate\n- Intent recognition accuracy\n- Tour booking conversion rate\n- Average conversation duration\n- Escalation rate to human agents\n\n### Regular Maintenance\n- Weekly performance review\n- Monthly knowledge base updates\n- Quarterly conversation flow optimization\n- Annual security and compliance audit\n\n## Support Resources\n\n### Documentation\n- [Retell.AI Documentation](https://docs.retell.ai)\n- [Voice Agent Best Practices](./Best%20Practices%20Retell%20knowledge%20base.md)\n- [Knowledge Base Template](./Knowledge%20Base%20Template.md)\n\n### Configuration Files\n- [Agent Configuration](./LeoVoiceAgent.json)\n- [Conversation Prompts](./PROMPT%20-%20Leo%20Voice%20Agent.md)\n- [Sample Knowledge Base](./Lakeside%20Apartments%20-%20MOCK%20DATA%20SET.md)\n\n### Development Support\n- Check `dev-reviews/retell-agent/issues/` for current development tasks\n- Review integration documentation in `project-docs/technical-architecture/`\n- Follow deployment procedures in `project-docs/project-planning/`\n\n---\n\n**The LEO voice agent is designed to provide intelligent, natural conversation experiences while efficiently capturing leads and scheduling tours for property management companies.**"
